# CLAUDE.md

This file guides Claude Code when working in the **teg-blue-overview** repository.

## Purpose

This is the public **overview / entry-point** repo for TEG-Blue on GitHub. It holds the high-level project map (`README.md`) that points newcomers to the four TEG-Blue layers and their repos. It does **not** hold framework definitions or any site's internal code.

- `README.md` — the GitHub-facing project map. Keep it current and public-safe.
- **Source of truth for definitions, ecosystem language, and claims is `what-is-teg-blue/`** (in the parent `Projects/` folder), not this repo. When updating the README, align with it.

## The Four Layers

| Layer | Repo / home | Role |
|---|---|---|
| Public framework / research | `teg-blue-site-org` → teg-blue.org | The Nervous System Gradient; framework, methodology, source traces, publications |
| TEG-Blue Engine | `teg-blue-engine` | Internal, potentially licensable tool-building logic (ATLAS / Formations Map is one view inside it) |
| Practical public tools | `teg-blue-site-com` → teg-blue.com | Interactive pattern-reading tools |
| Applied / licensed builds | e.g. `violentometro-2-0` | Context-specific builds; IPN / Dra. Martha boundary, not signed yet |

## When to update the README

- A repo is added, renamed, or changes layer
- The four-layer model or core public claims change
- Keep research posture safe: **research-grounded / source-traceable**, never "validated as a whole," "diagnoses people," or "infers internal states" (exact language guardrails live in `what-is-teg-blue/`)

For the full cross-project folder map, see `Projects/CLAUDE.md`. For definitions, see `Projects/what-is-teg-blue/`.
